# 추천 지식 설계 문서

## 1. 문서 목적
이 문서는 `Alcoholic-BE`의 안주 추천 기능에 들어가는 지식 설계 원칙을 정리한 문서다.

나중에 다음 작업에 바로 재사용할 수 있도록 남긴다.

- 보고서/발표 자료 작성
- LLM 프롬프트 설계
- 레시피 seed 데이터 확장
- 추천 로직 고도화
- 사람 검수 기준 통일

## 2. 현재 추천 시스템 방향
현재 추천 시스템은 **DB 우선 + 점수화 + 필요 시 LLM 보완** 구조를 목표로 한다.

즉 운영 중 추천은 아래 순서로 동작한다.

1. 감지된 주류를 입력으로 받는다.
2. 현재 재고를 조회한다.
3. DB에 저장된 레시피 후보를 주류 기준으로 조회한다.
4. 현재 재고와 후보 레시피 재료를 비교한다.
5. 부족 재료 수와 사용 가능한 재료 수를 기준으로 점수화한다.
6. 상위 3개를 추천한다.
7. 나중에 필요하면 LLM으로 추천 이유를 더 자연스럽게 다듬거나 후보를 확장한다.

핵심은 다음과 같다.

- 추천 실행 시점에는 DB를 기준으로 움직인다.
- LLM은 운영 중 매번 전체 추천을 생성하는 주체가 아니다.
- LLM은 후보 데이터 구축과 설명 보완 역할로 쓴다.

## 3. 현재 DB seed 운영 기준
현재 seed는 `seeds/recommendations.json`에 저장한다.

현재 1차 목표 규모:

- 주류 7종
- 각 주류당 10개 레시피
- 총 70개

주류 카테고리:

- `soju`
- `beer`
- `white_wine`
- `red_wine`
- `whisky`
- `sparkling_wine`
- `sake`

각 레시피는 현재 아래 필드를 가진다.

- `liquor_name`
- `refresh_group`
- `rank_hint`
- `name`
- `reason`
- `recipe`
- `ingredients`

## 4. pantry(상온 양념) 가정
추천 시스템은 **상온 양념 재료는 기본 보유**한다고 가정한다.

예시:

- 소금
- 후추
- 간장
- 식용유
- 식초
- 설탕
- 고춧가루
- 기본 소스류

따라서 `missing_ingredients`에는 되도록 아래처럼 **냉장고/보관 재료로 체감되는 핵심 재료** 위주로 들어가야 한다.

- 육류/생선
- 달걀
- 채소
- 유제품
- 소시지/베이컨
- 빵

즉 seed의 `ingredients`는 "없으면 실제로 냉장고에서 꺼내야 하는 재료" 중심으로 유지한다.

## 5. 추천 품질 검수 기준
레시피를 `approved` 수준으로 보기 위한 핵심 기준은 아래와 같다.

1. 해당 주류와 실제로 어울리는가
2. 한국 사용자 기준으로 납득되는가
3. 집에서 만들 수 있는 난이도인가
4. 현재 프로젝트 식재료 클래스와 연결 가능한가
5. 추천 이유가 사용자가 이해할 수 있을 만큼 설득력 있는가
6. 추천 3개가 서로 지나치게 비슷하지 않은가

검수 판단 기준:

- `유지`: 그대로 써도 무난함
- `수정`: 방향은 맞지만 더 좋은 대체안이 있음
- `제외`: 페어링 설득력이 약하거나 서비스 톤과 맞지 않음

## 6. 주류별 페어링 원칙

### 6-1. soju
기본 방향:

- 짭짤함
- 볶음
- 전
- 무침
- 돼지고기/소시지/달걀/대파 같은 익숙한 한식형 안주

설명 원칙:

- 소주의 직선적인 맛을 눌러주거나
- 느끼함을 잡아주는 조합을 강조한다.

### 6-2. beer
기본 방향:

- 감자
- 치즈
- 베이컨
- 소시지
- 닭고기
- 토스트/구이류

설명 원칙:

- 탄산감과 짭짤함, 고소함, 바삭함의 궁합을 강조한다.

### 6-3. white_wine
기본 방향:

- 생선
- 닭고기
- 치즈
- 요거트
- 가벼운 샐러드
- 버터를 곁들인 담백한 조리

설명 원칙:

- 산미
- 담백함
- 부드러운 질감

이 세 가지의 균형을 강조한다.

### 6-4. red_wine
기본 방향:

- 소고기
- 버섯
- 토마토
- 치즈
- 가지
- 베이컨

설명 원칙:

- 육향
- 버섯의 우마미
- 토마토 산미
- 진한 풍미

처럼 더 묵직한 조합을 강조한다.

주의:

- 너무 가벼운 샐러드형 메뉴는 우선순위를 낮춘다.

### 6-5. whisky
기본 방향:

- 베이컨
- 소고기
- 소시지
- 치즈
- 버터
- 버섯

설명 원칙:

- 훈연감
- 짭짤함
- 묵직함
- 진한 풍미

와 연결한다.

주의:

- 너무 산뜻하거나 채소 중심인 메뉴는 우선순위를 낮춘다.

### 6-6. sparkling_wine
기본 방향:

- 한입 크기 메뉴
- 샐러드
- 치즈
- 오이/토마토
- 가벼운 토스트
- 산뜻한 조합

설명 원칙:

- 청량감
- 산뜻함
- 가벼운 치즈/채소 조합

을 강조한다.

주의:

- 너무 묵직하고 기름진 메뉴는 우선순위를 낮춘다.

### 6-7. sake
기본 방향:

- 생선
- 달걀
- 버섯
- 대파
- 닭고기
- 담백하고 우마미 있는 조합

설명 원칙:

- 섬세한 향
- 우마미
- 자극적이지 않은 조리

를 강조한다.

주의:

- 토마토 산미가 강하거나 너무 서양식으로 치우친 조합은 우선순위를 낮춘다.

## 7. 2026-04-24 1차 큐레이션 변경 내용
이번 1차 검수에서 특히 수정 우선순위가 높았던 카테고리는 아래 4개였다.

- `red_wine`
- `whisky`
- `sparkling_wine`
- `sake`

교체한 항목:

- `red_wine`
  - `치즈 브로콜리그라탱풍 볶음` -> `버섯 치즈구이`
  - `토마토 양파샐러드` -> `베이컨 토마토볶음`
- `whisky`
  - `베이컨 브로콜리볶음` -> `베이컨 양파볶음`
  - `브레드 치즈플레이트풍 토스트` -> `소시지 치즈구이`
- `sparkling_wine`
  - `감자 치즈바이트` -> `치즈 오이 카나페풍 토스트`
  - `요거트 토마토브레드` -> `토마토 치즈 카나페풍 토스트`
- `sake`
  - `양배추 닭볶음` -> `닭고기 버섯볶음`
  - `생선 토마토조림풍 볶음` -> `생선 버섯구이풍 볶음`
  - `당근 양배추볶음` -> `양배추 버섯볶음`

수정 이유는 공통적으로 아래에 가깝다.

- 주류와의 거리감이 더 가까운 메뉴로 교체
- 너무 가벼운/어색한 메뉴 제거
- 서비스 설명력이 더 높은 메뉴로 교체

## 8. 앞으로의 운영 방식
현재 seed는 `approved` 후보 집합처럼 다루되, 장기적으로는 아래 상태 관리가 있으면 좋다.

- `draft`: LLM 초안
- `reviewed`: 사람이 1차 검토
- `approved`: 서비스 노출 가능
- `archived`: 제외

향후 1000개 규모까지 확장할 때는 반드시 위 상태 관리가 필요하다.
